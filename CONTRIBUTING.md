# Contributing

Thank you for your interest in this project.

This repository is maintained by the Ministry of Justice and developed in the open in accordance with UK Government and MOJ guidance on publishing source code. While contributions are welcome, this is not a community-driven project and development priorities are determined by the maintainers. 

## Before opening a Pull Request

Please open an Issue to discuss any significant change before investing time in implementation.

This allows the maintainers to:

- Determine whether the proposed change aligns with the project's roadmap
- Avoid duplicate or conflicting work
- Consider operational, security, accessibility and support implications
- Provide guidance on the preferred approach

Pull requests may be closed if they relate to changes that have not been discussed beforehand or are not aligned with the project's priorities.

Opening an Issue does not guarantee that a proposal will be accepted, but it helps ensure that effort is spent on changes that are likely to be valuable to both contributors and maintainers.

## Reporting Bugs

If you discover a bug, please open an Issue and include:

- A clear description of the problem
- Steps to reproduce the issue
- Expected behaviour
- Actual behaviour
- Any relevant logs or screenshots

> [!WARNING]
> Do **not** include screenshots, logs, exported data, or other attachments that contain:
>
> - Personal information
> - Offender information
> - Staff information
> - Email addresses
> - Usernames
> - Access tokens, API keys, or credentials
> - Internal URLs, hostnames, or infrastructure details
> - Any information that is not already publicly available
>   
> All issues in this repository are publicly visible. Before submitting an issue, ensure that any screenshots or attachments have been reviewed and appropriately redacted.


## Suggesting Enhancements

We welcome suggestions for improving the project.

Please raise an Issue describing:

- The problem you are trying to solve
- Your proposed approach
- Any alternative solutions you have considered

The maintainers will review the proposal and decide whether it fits within the scope and priorities of the project.

## Security Vulnerabilities

Please do not report security vulnerabilities through public GitHub Issues.

Follow the Ministry of Justice security reporting guidance instead. [1](https://github.com/ministryofjustice)

## Maintainer Responsibility

The maintainers are responsible for:

- Setting project direction and priorities
- Reviewing and accepting contributions
- Ensuring ongoing support and maintainability

Not all proposed changes will be accepted, even after discussion.
